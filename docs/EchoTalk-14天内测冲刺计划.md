# EchoTalk 首次内测版本 · 14 天冲刺计划

## 范围与前提

| 项 | 内容 |
|---|---|
| 内测功能 | 影子跟读、账号体系、支付与会员（**模拟支付**） |
| 涉及端 | 客户端 App（Flutter）、后台（Go）、管理端（Vue） |
| 平台 | **仅 Android** |
| 开发模式 | 单人全职，约 8 小时/天 |
| 技术储备 | Go 后端已熟；Flutter、Vue 边学边做 |
| 支付口径 | 个人开发者无营业执照、内测不上 iOS → **无真实支付通道**；会员/付费墙系统做真，交易走**模拟通道（适配器模式）** |
| 唯一外部审批依赖 | 讯飞 ISE（语音评测），**Day 1 即申请** |

**两条贯穿始终的设计约束（现在就做对，避免返工）：**
- **支付适配器模式**：支付通道抽象为接口，内测实现 `MockChannel`；日后接微信/支付宝/IAP 只新增实现，会员逻辑不动。
- **付费边界服务端校验**：免费/付费解锁判断必须在后端，客户端只做展示。

## 第一阶段：后端核心（Day 1–5）

### Day 1 — 基建与项目骨架
- **目标**：三仓与服务器开发链路打通，数据模型定稿，外部审批启动。
- **涉及**：基建 / 后端。
- **任务分解**：
  - [ ] 建三个 Git 仓库 `backend` / `app` / `admin`，各放一份聚焦的 CLAUDE.md
  - [ ] 服务器用 Portainer 起栈（docker-compose）：Nginx + MySQL 8 + Redis + Go 占位服务
  - [ ] 创建 COS 桶 + 绑定 CDN 加速域名 + 配置防盗链
  - [ ] 设计并提交 OpenAPI 契约骨架（账号/内容/训练/会员接口签名）
  - [ ] 设计数据库 schema：`users` `videos` `sentences` `training_records` `products` `orders` `memberships`（含免费/付费字段）
  - [ ] 后端脚手架：gin + viper（配置）+ GORM（迁移）+ zap（日志）+ 统一响应/错误码 + 健康检查
  - [ ] **并行**：申请讯飞 ISE 语音评测服务；后台启动域名备案（内测可暂缓，但早启动）
  - [ ] 当晚：安装 Flutter SDK + 跑通 Android 真机/模拟器调试链路（提前进入学习）
- **完成标准**：
  - 服务器上 `docker-compose up` 一键起栈，Portainer 见 4 容器健康
  - `GET /health` 返回 200；GORM 自动建表成功，能连 MySQL/Redis
  - OpenAPI 骨架可在 Swagger UI 打开
  - COS 能上传一个文件并经 CDN 域名访问
- **产出/依赖**：三仓骨架、容器栈、schema 迁移、OpenAPI v0、讯飞申请单号。

### Day 2 — 账号体系（后端）
- **目标**：注册/登录/鉴权闭环可用。
- **涉及**：后端。
- **任务分解**：
  - [ ] 注册：邮箱（手机号占位）+ 固定验证码（内测）写入 `users`
  - [ ] 登录：校验 → 签发 JWT（access + refresh）
  - [ ] 鉴权中间件：解析/校验 token，注入用户上下文
  - [ ] token 刷新接口；登出（可选）
  - [ ] 密码 bcrypt 加密、validator 参数校验
- **完成标准**：
  - curl 走通：注册 → 登录拿 token → 带 token 访问受保护接口成功，无 token 返回 401
  - token 过期后用 refresh 换新成功
- **产出/依赖**：账号模块接口 + curl/Postman 用例。

### Day 3 — 内容与媒体（后端）
- **目标**：App 能取到内容与句级数据，媒体经 CDN 播放。
- **涉及**：后端。
- **任务分解**：
  - [ ] `videos` CRUD（内部）+ `sentences` 模型：起止时间戳、英文文本、可选译文/音标、免费/付费标记
  - [ ] App 侧接口：内容列表（含付费锁标识）、内容详情（含句级数组）
  - [ ] COS 上传（服务端签名直传或代传）+ 返回 CDN 播放地址
  - [ ] Go 导入命令：把「视频元数据 + 句级 JSON」一键入库
  - [ ] 录入 1 条真实样例内容（含字幕时间戳）
- **完成标准**：
  - 导入命令成功入库 1 条带 N 句的内容
  - 列表接口标出免费/付费；详情接口返回完整句级数组
  - 用返回的 CDN 地址在浏览器/VLC 能播放 HLS
- **产出/依赖**：内容/媒体接口、导入命令、1 条可播放样例。

### Day 4 — 语音评测接入（后端）
- **目标**：上传录音即可得到讯飞评分。
- **涉及**：后端（语音能力网关）。
- **任务分解**：
  - [ ] 语音网关封装讯飞 ISE 云端 API：`read_sentence`（英文）、HMAC 鉴权、试题文本参数、音频 16K/16bit/单声道 WAV
  - [ ] 评测接口：App 上传录音（+句子文本/句ID）→ 网关 → 返回总分及发音/流利度/完整度/词级
  - [ ] `training_records` 落库；训练历史接口
  - [ ] 讯飞错误码处理、超时降级（不抛 500）
- **完成标准**：
  - curl 上传一段 16K WAV → 返回结构化评分 JSON 并落库
  - 训练历史接口能列出该用户记录
  - 讯飞异常时接口有兜底返回
- **产出/依赖**：评测接口、训练记录/历史接口、语音网关。依赖讯飞 ISE 已开通（未就绪则见降级预案）。

### Day 5 — 会员与支付（后端 · 模拟通道）
- **目标**：下单 → 模拟支付 → 开会员 → 付费内容放行 全链路（服务端）。
- **涉及**：后端。
- **任务分解**：
  - [ ] `products`（SKU/价格/会员时长）+ `orders` + `memberships` 模型
  - [ ] **支付适配器接口 `PaymentChannel`** + 实现 `MockChannel`（下单→标记成功）
  - [ ] 下单接口 → 创建待支付订单
  - [ ] 模拟支付确认 → 订单置成功 → 写/续 `memberships`
  - [ ] **内容付费边界服务端校验**：付费内容接口对非会员拒绝、对会员放行
  - [ ] 手动发卡接口（运营给指定用户开会员）
- **完成标准（M1 里程碑）**：
  - curl 全链路：下单 → 模拟支付成功 → 用户会员有效 → 付费内容接口由 403 转放行
  - 非会员访问付费内容被**服务端**拒绝（非仅前端隐藏）
  - 适配器接口清晰，新增真实通道无需改会员逻辑
- **产出/依赖**：会员/订单/模拟支付接口、适配器骨架、付费门禁。

## 第三阶段：客户端 App（Day 8–12 · 仅 Android）

### Day 8 — Flutter 脚手架 + 账号
- **目标**：App 能登录并保持会话。
- **涉及**：App。
- **任务分解**：

  - [ ] 鉴权态全局管理（已登录/未登录路由守卫）

  - token 过期自动刷新，刷新失败跳登录
- **产出/依赖**：可登录的 App 骨架。

- **目标**：能浏览内容并播放视频。
- **涉及**：App。
- **任务分解**：
  - [ ] 内容列表页（展示 + 付费锁标识）+ 详情页
  - [ ] 播放器接入（better_player / video_player，HLS）
- **完成标准**：
  - 列表正确展示免费/付费标识，点击进入详情
  - 视频在真机正常播放（HLS）

### Day 10 — 影子跟读主体
- **目标**：逐句跟读 + 录音 + 评分跑通。
- **涉及**：App。
- **任务分解**：
  - [ ] 按句导航：依时间戳定位句首/句尾、当前句循环播放
  - [ ] 字幕随播放进度同步高亮
  - [ ] 逐句录音（record，16K/16bit/单声道 WAV）+ 上传后端评测接口
  - [ ] 展示评分（发音/流利度/完整度）+ 原声 vs 录音对照回放
- **完成标准**：
  - 选一句 → 听原声 → 录音 → 上传 → 显示讯飞评分 → 可回放对照
  - 字幕高亮与播放进度同步
- **产出/依赖**：影子跟读核心交互（单句闭环）。依赖 Day 4 评测接口。

### Day 11 — 付费墙 + 模拟购买 + 解锁
- **目标**：付费内容的购买—解锁体验跑通。
- **涉及**：App。
- **任务分解**：
  - [ ] 访问付费内容时弹付费墙（展示 SKU）
  - [ ] 模拟购买：调后端下单 + 模拟支付 → 刷新会员态 → 解锁
  - [ ] 会员状态展示（个人中心）
- **完成标准（M3 里程碑）**：
  - 非会员点付费内容 → 付费墙 → 模拟购买成功 → 内容解锁可跟读
  - 个人中心显示会员有效期
- **产出/依赖**：付费墙 + 会员闭环（模拟）。依赖 Day 5/7。

### Day 12 — 训练历史 + 打磨 + 缓冲
- **目标**：补齐历史、打磨主流程、吸收学习延误。

  - [ ] 训练历史页（成绩列表/单次详情）
  - [ ] 主流程异常态/空态/loading 处理
  - [ ] **机动缓冲**：消化 Flutter 学习中的延误
- **完成标准**：
  - 历史页能看到过往评测记录
